The ask: Build a new elementary/middle school simulation software for a physical, hands-on experience where students operate businesses, manage money, and fulfil civic roles in a simulated town for a day. The new platform replaces the existing system with a modern, tablet-first, real-time experience.
What we're building

Core Platform

A unified digital platform supporting a real-economy simulation — banking, payroll, point-of-sale, voting, marketing, and civic engagement — running simultaneously across 80–150 student devices in a physical facility.

Key Design Principle

Real Feels Real

Tap-to-pay, live dashboards, AI-assisted marketing, digital banking — the UX should mirror tools students will encounter in the real world, but age-appropriate and guided.

Economic Model — How the Town Works
Businesses (14–20)

Each storefront has 6–8 student employees. Apply for loans, manage payroll, run POS, serve customers, send/receive B2B invoices.

Banking & Finance

Students hold personal checking + savings accounts. Debit cards, QR-code payments, NFC tap-to-pay. Real-time balance sync across all devices.

Civic Layer

Voting on town issues, tax mechanics, philanthropic scenarios, community leaderboard — students see how personal decisions affect the whole economy.

Learning Objectives
Delivery Modes
Permanent Facility

Full onsite JA BizTown installation in a dedicated physical space.

Mobile / Pop-Up

Tablet-based "JA BizTown Mobile" deployed in schools, community centres.

Paper Fallback

Required: complete paper-based version for power/internet outages.

Section 04
Required Software Modules
Every module below is a hard requirement. The platform must support all simultaneously on a one-device-per-student model with real-time data sync across all 80–150 devices.
💳
Point of Sale System
Set sale price per storefront. Profit/loss slider based on loan vs. revenue. Real-time transaction recording.
📲
Student Payment
QR code per device as payment. Colour-coded shopping break state. Debit card swiper support + NFC tap-to-pay.
🎨
Digital Marketing Tool
Canva-style tool built into software. Share designs between devices, roles, businesses. AI slogan generation.
🏦
Banking System
Personal checking + savings accounts. Deposits, withdrawals, real-time balance updates across all systems.
💰
Payroll System
Direct deposit or printed paychecks. 2nd paycheck as triggered spot bonus. Simulation lead controls timing.
📋
Loan Application & Approval
CFO applies on shared screen visible to all team. CEO finalises at bank. Screen-share view for team members.
📬
Billing & Bill Pay
Send and receive invoices between businesses. B2B transaction tracking. Overdue bill alerts.
🗳️
Voting
Kiosk version + mobile version for Mayor to carry. Results feed into leaderboard and State of the Town.
🌪️
Business Scenarios
Randomised ethical scenarios (e.g. tornado hits town park). CEO decides contribution. XP gain/loss consequences.
🏆
Leaderboard (XP)
Live, presentable on large screen. Scores: loan payoff speed, profit, invoices paid, donations, surveys, voting.
📊
Business Dashboard
P&L statements, profit calculator, competitor pricing view, loan amount, pie charts + bar graphs.
📻
Radio Business
Music/radio software integrated with facility A/V. Auto-imports business slogans for DJ to broadcast.
📝
Surveys & Evaluation
Pre + post. Lock 4–6 questions nationally. Areas add up to 6. Multiple choice, yes/no, free response. Min 10 items, 125-char prompts.
🤝
Team Building Widget
One-device-per-user real-time collaborative widget. Each student's input visibly contributes to a shared business output.
🤖
AI Features
Slogan generation for marketing. CEO speech transcription via voice-to-text tablet input.

Section 05
Audience Roles & Requirements
Role Summary
Students — Primary Users
  • Grades 5–6, ages 9–12
  • One device per student (tablet)
  • Roles: CEO, CFO, CMO, general employees
  • Apply for loans, submit payroll, vote, shop
  • Authentication: QR code or PIN — no device lock-in
  • English + Spanish UI at minimum
Simulation Manager (Staff)
  • Full-time JA Area staff, leads entire day on mic
  • Centralized dashboard: real-time town status
  • Can push/pause simulation phases from laptop
  • Admin tools: reverse transactions, reissue paychecks, move students between businesses remotely
  • Award XP and subsidies to businesses
  • Trigger activities (voting, scenarios, payroll)
Volunteers
  • Corporate, parent, or community volunteers
  • Tablet dashboard: business P&L, student progress, transactions, loan status
  • Integrated volunteer guide with discussion prompts
  • Clear indicators for when + why to intervene
  • End-of-day feedback survey
Teachers
  • Deliver 12 pre-sim lessons before simulation day
  • Monitor student behaviour and engagement day-of
  • Access end-of-day reports via QR code or simple login
  • Individual student reports, business reports, full simulation report, voting results, evaluation data
  • May serve as improvised volunteer if needed
CMS Admin Roles
Super Admin (National)

JA USA. Sets national master templates, locks evaluation questions, pushes updates to all JA Areas without overwriting local customisation.

Local Admin

Per JA Area. Customises businesses, sponsors, logos, payroll rates, tax rates, loan terms. Can only access facilities they're assigned to.

Simulation Manager

Day-of setup and activation. Creates simulation templates, adjusts timing, enables/disables modules, manages real-time CMS changes during live sim.

Non-Negotiable Requirements
Accessibility

WCAG 2.2 AA compliance mandatory. VPAT required for each section. English + Spanish at minimum. Visual, text, and optional read-aloud modes.

Real-Time Sync

Every transaction must update simultaneously across student account, business account, and financial institution — no lag, no discrepancy.

Security

IETF, OWASP, and W3C standards. Student PII protected. Data encrypted at rest and in transit. Full audit logging in CMS and dashboards.

Agile / Scrum

Standard sprint cadence. Scrum ceremonies: pre-sprint planning, standups, refinement, retros, review/demo. Work items in Azure DevOps.

Paper Fallback

Complete paper-based version of simulation required for power/internet outage scenarios. CMS generates as PDFs.

Localisation

All text + alt text accessible by browser translation. XLF files for English + Spanish. Expandable text containers for character count variation.
